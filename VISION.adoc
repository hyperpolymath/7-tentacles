*A Complete Pedagogical Framework for Teaching Compiler Architecture*

== Executive Summary

Seven Tentacles is a *radically innovative 10-year learning framework* that teaches university and graduate-level compiler architecture concepts to children aged 8-18 without them realizing it.

=== The Core Insight

Kids learn by playing games with colorful mascots. At age 8, they meet "Speedy Red" and play racing games. At age 16, they discover that "Speedy Red" has been teaching them about recursive descent parsing, and Red now goes by "Performance Agent Red." They've been learning compilers for 8 years without noticing.

=== The Stack

* **7 cephalopod mascots** (octopus agents) — each teaches one compiler component
* **4 language dialects** (Me, Solo, Duet, Ensemble) — matching their age/maturity
* **Progressive reveal system** — concepts introduced as games, revealed as CS
* **10-year narrative journey** — Cuttle → Squidlet → Duet → Octopus

=== Target Outcome

16-year-olds with *graduate-level understanding* of:

* Parsing & lexical analysis
* Concurrency & async/await
* Type systems & affine types
* AST construction & manipulation
* Audit trails & verification
* Compile-time evaluation
* Language governance & policy enforcement

== Language Progression: Me → Solo → Duet → Ensemble

Each dialect of "Me Language" matches the current tentacle stage.

=== Me Language (Ages 8-12) — Visual/Blockly

[source]
----
// Blocks, no syntax
[Red] do [race]
[Orange] do [juggle] [Red result]
[Yellow] check [all types valid]
[show] [result on screen]
----

*Features:*

* No text syntax
* Drag-and-drop blocks
* Visual representation of data flow
* One agent at a time (mostly)
* Immediate visual feedback

=== Solo Language (Ages 13-14) — Affine Text

[source,rust]
----
fun race(distance: Meters) -> Time {
  Red.optimize(distance) |> Racer.run
}

fun sort_game(items: List[Item]) -> List[Item] {
  Yellow.check_types(items);
  items.sort_by(|x| x.speed)
}
----

*Features:*

* Text-based, clean syntax
* Affine types (resources, ownership)
* Single agent/function focus
* Gradual type inference
* Error messages mention agent names

=== Duet Language (Ages 15) — Two-Party Protocols

[source,rust]
----
protocol DuetDance {
  role Red {
    send parsed_ast to Yellow
    wait for validation_result
  }

  role Yellow {
    receive ast from Red
    verify_all_constraints(ast)
    send validation_result to Red
  }
}

let result = DuetDance::run(my_code)
----

*Features:*

* Protocol-oriented, explicit interaction
* Two agents coordinate
* Session types ensure correctness
* Error messages explain protocol violations
* Visual representation of message flow

=== Ensemble Language (Ages 16+) — Full Orchestration

[source,rust]
----
ensemble Compiler {
  agent red: Red
  agent orange: Orange
  agent yellow: Yellow
  agent green: Green
  agent blue: Blue
  agent indigo: Indigo
  agent violet: Violet

  pipeline {
    let tokens = red.lex(source_code)
    let ast = red.parse(tokens)
    yellow.check_types(ast)?
    let ir = green.construct(ast)
    let optimized = indigo.specialize(ir)
    let machine_code = red.codegen(optimized)
    blue.verify_all(machine_code)?
    violet.approve_execution(machine_code)?
    machine_code
  }
}
----

*Features:*

* 7 agents coordinate
* Explicit pipeline / orchestration
* Error handling with verification
* Formal guarantees via Blue & Violet
* Full compiler architecture visible
